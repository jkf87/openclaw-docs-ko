OpenClaw 위협 모델 v1.0 ​

MITRE ATLAS 프레임워크 ​

버전: 1.0-draft 최종 업데이트: 2026-02-04 방법론: MITRE ATLAS + 데이터 흐름 다이어그램 프레임워크: MITRE ATLAS (AI 시스템을 위한 적대적 위협 환경)

프레임워크 귀속 ​

이 위협 모델은 AI/ML 시스템에 대한 적대적 위협을 문서화하기 위한 산업 표준 프레임워크인 MITRE ATLAS를 기반으로 합니다. ATLAS는 AI 보안 커뮤니티와 협력하여 MITRE가 관리합니다.

주요 ATLAS 리소스:

  • ATLAS 기법
  • ATLAS 전술
  • ATLAS 사례 연구
  • ATLAS GitHub
  • ATLAS에 기여하기

이 위협 모델에 기여하기 ​

이것은 OpenClaw 커뮤니티가 유지하는 살아있는 문서입니다. 기여 가이드라인은 CONTRIBUTING-THREAT-MODEL.md를 참조하십시오:

  • 새 위협 보고
  • 기존 위협 업데이트
  • 공격 체인 제안
  • 완화 방법 제안

1. 소개 ​

1.1 목적 ​

이 위협 모델은 AI/ML 시스템을 위해 특별히 설계된 MITRE ATLAS 프레임워크를 사용하여 OpenClaw AI 에이전트 플랫폼과 ClawHub 스킬 마켓플레이스에 대한 적대적 위협을 문서화합니다.

1.2 범위 ​

컴포넌트포함 여부참고
OpenClaw 에이전트 런타임예핵심 에이전트 실행, 도구 호출, 세션
게이트웨이예인증, 라우팅, 채널 통합
채널 통합예WhatsApp, Telegram, Discord, Signal, Slack 등
ClawHub 마켓플레이스예스킬 게시, 조정, 배포
MCP 서버예외부 도구 프로바이더
사용자 기기부분모바일 앱, 데스크톱 클라이언트

1.3 범위 외 ​

이 위협 모델에서 명시적으로 범위 외인 것은 없습니다.

3. ATLAS 전술별 위협 분석 ​

3.1 정찰 (AML.TA0002) ​

T-RECON-001: 에이전트 엔드포인트 검색 ​

속성값
ATLAS IDAML.T0006 - 능동 스캔
설명공격자가 노출된 OpenClaw 게이트웨이 엔드포인트를 스캔합니다
공격 벡터네트워크 스캔, shodan 쿼리, DNS 열거
영향 받는 컴포넌트게이트웨이, 노출된 API 엔드포인트
현재 완화 방법Tailscale 인증 옵션, 기본적으로 루프백에 바인딩
잔류 위험중간 - 공개 게이트웨이 검색 가능
권고 사항안전한 배포 문서화, 검색 엔드포인트에 속도 제한 추가

T-RECON-002: 채널 통합 프로빙 ​

속성값
ATLAS IDAML.T0006 - 능동 스캔
설명공격자가 메시징 채널을 프로빙하여 AI 관리 계정을 식별합니다
공격 벡터테스트 메시지 전송, 응답 패턴 관찰
영향 받는 컴포넌트모든 채널 통합
현재 완화 방법특별한 것 없음
잔류 위험낮음 - 검색만으로는 가치 제한적
권고 사항응답 타이밍 무작위화 고려

공격 벡터어깨너머 보기, 네트워크 스니핑, 사회공학
영향 받는 컴포넌트기기 페어링 시스템
현재 완화 방법1시간 만료 (DM 페어링) / 5분 만료 (노드 페어링), 기존 채널을 통해 코드 전송
잔류 위험중간 - 유예 기간 악용 가능
권고 사항유예 기간 단축, 확인 단계 추가

T-ACCESS-002: AllowFrom 스푸핑 ​

속성값
ATLAS IDAML.T0040 - AI 모델 추론 API 액세스

공격 벡터채널에 따라 다름 - 전화번호 스푸핑, 사용자 이름 사칭
영향 받는 컴포넌트채널별 AllowFrom 유효성 검사
현재 완화 방법채널별 아이덴티티 확인
잔류 위험중간 - 일부 채널이 스푸핑에 취약
권고 사항채널별 위험 문서화, 가능한 경우 암호화 검증 추가

T-ACCESS-003: 토큰 도용 ​

속성값
ATLAS IDAML.T0040 - AI 모델 추론 API 액세스
설명공격자가 구성 파일에서 인증 토큰을 도용합니다
공격 벡터맬웨어, 무단 기기 액세스, 구성 백업 노출
영향 받는 컴포넌트~/.openclaw/credentials/, 구성 저장소
현재 완화 방법파일 권한

권고 사항저장된 토큰 암호화 구현, 토큰 교체 추가

3.3 실행 (AML.TA0005) ​

T-EXEC-001: 직접 프롬프트 주입 ​

속성값
ATLAS IDAML.T0051.000 - LLM 프롬프트 주입: 직접
설명공격자가 에이전트 동작을 조작하는 프롬프트를 전송합니다
공격 벡터적대적 지시를 포함하는 채널 메시지
영향 받는 컴포넌트에이전트 LLM, 모든 입력 인터페이스
현재 완화 방법패턴 감지, 외부 콘텐츠 래핑
잔류 위험심각 - 감지만 가능, 차단 없음; 정교한 공격은 우회 가능
권고 사항다중 레이어 방어 구현, 출력 유효성 검사, 민감한 작업에 사용자 확인

T-EXEC-002: 간접 프롬프트 주입 ​

속성값
ATLAS IDAML.T0051.001 - LLM 프롬프트 주입: 간접
설명공격자가 가져온 콘텐츠에 악의적 지시를 삽입합니다

권고 사항콘텐츠 정화 구현, 별도의 실행 컨텍스트

T-EXEC-003: 도구 인수 주입 ​

속성값
ATLAS IDAML.T0051.000 - LLM 프롬프트 주입: 직접

권고 사항인수 유효성 검사 구현, 매개변수화된 도구 호출

T-EXEC-004: Exec 승인 우회 ​

속성값

설명공격자가 승인 허용 목록을 우회하는 명령을 제작합니다
공격 벡터명령 난독화, 별칭 악용, 경로 조작
영향 받는 컴포넌트exec-approvals.ts, 명령 허용 목록
현재 완화 방법허용 목록 + ask 모드
잔류 위험높음 - 명령 정화 없음
권고 사항명령 정규화 구현, 차단 목록 확장

영향 받는 컴포넌트ClawHub, 스킬 로딩, 에이전트 실행

권고 사항VirusTotal 통합 (진행 중), 스킬 샌드박싱, 커뮤니티 검토

T-PERSIST-002: 스킬 업데이트 오염 ​

속성값
ATLAS IDAML.T0010.001 - 공급망 손상: AI 소프트웨어
설명공격자가 인기 있는 스킬을 손상시키고 악의적 업데이트를 푸시합니다
공격 벡터계정 손상, 스킬 소유자에 대한 사회공학
영향 받는 컴포넌트ClawHub 버전 관리, 자동 업데이트 흐름
현재 완화 방법버전 지문
잔류 위험높음 - 자동 업데이트가 악의적 버전을 가져올 수 있음
권고 사항업데이트 서명 구현, 롤백 기능, 버전 고정

T-PERSIST-003: 에이전트 구성 변조 ​

속성값
ATLAS IDAML.T0010.002 - 공급망 손상: 데이터

권고 사항구성 무결성 확인, 구성 변경에 대한 감사 로깅

4. ClawHub 공급망 분석 ​

4.1 현재 보안 제어 ​

제어구현효과
GitHub 계정 연령requireGitHubAccountAge()중간 - 새 공격자에 대한 장벽 높임
경로 정화sanitizePath()높음 - 경로 순회 방지
파일 유형 유효성 검사isTextFile()중간 - 텍스트 파일만, 하지만 여전히 악의적일 수 있음
크기 제한50MB 총 번들높음 - 자원 고갈 방지
필수 SKILL.md필수 README낮음 보안 가치 - 정보 제공만
패턴 조정moderation.ts의 FLAG_RULES낮음 - 쉽게 우회됨
조정 상태moderationStatus 필드중간 - 수동 검토 가능

4.2 조정 플래그 패턴 ​

moderation.ts의 현재 패턴:

javascript
// 알려진 나쁜 식별자
/(keepcold131\/ClawdAuthenticatorTool|ClawdAuthenticatorTool)/i

// 의심스러운 키워드
/(malware|stealer|phish|phishing|keylogger)/i
/(api[-_ ]?key|token|password|private key|secret)/i
/(wallet|seed phrase|mnemonic|crypto)/i
/(discord\.gg|webhook|hooks\.slack)/i
/(curl[^\n]+\|\s*(sh|bash))/i
/(bit\.ly|tinyurl\.com|t\.co|goo\.gl|is\.gd)/i

제한 사항:

  • 슬러그, displayName, 요약, 프론트매터, 메타데이터, 파일 경로만 확인
  • 실제 스킬 코드 콘텐츠를 분석하지 않음
  • 단순 regex는 난독화로 쉽게 우회됨
  • 행동 분석 없음
