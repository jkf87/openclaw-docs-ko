# 네트워크 모델

> 이 내용은 [네트워크](/network#core-model)에 통합되었습니다. 현재 가이드는 해당 페이지를 참조하십시오.

대부분의 작업은 채널 연결과 WebSocket 컨트롤 플레인을 소유하는 단일 장시간 실행 프로세스인 게이트웨이(`openclaw gateway`)를 통해 흐릅니다.

## 핵심 규칙

- 호스트당 하나의 게이트웨이를 권장합니다. 이는 WhatsApp Web 세션을 소유할 수 있는 유일한 프로세스입니다. 복구 봇이나 엄격한 격리를 위해서는 격리된 프로필과 포트로 여러 게이트웨이를 실행합니다. [멀티 게이트웨이](/gateway/multiple-gateways) 참조.
- 루프백 우선: 게이트웨이 WS는 기본적으로 `ws://127.0.0.1:18789`입니다. 마법사는 기본적으로 공유 시크릿 인증을 생성하며, 루프백의 경우에도 일반적으로 토큰을 생성합니다. 비루프백 액세스를 위해서는 유효한 게이트웨이 인증 경로를 사용하십시오: 공유 시크릿 토큰/비밀번호 인증, 또는 올바르게 구성된 비루프백 `trusted-proxy` 배포. Tailnet/모바일 설정은 원시 tailnet `ws://` 대신 Tailscale Serve 또는 다른 `wss://` 엔드포인트를 통해 가장 잘 작동합니다.
- 노드는 필요에 따라 LAN, tailnet, 또는 SSH를 통해 게이트웨이 WS에 연결합니다. 레거시 TCP 브리지는 제거되었습니다.
- 캔버스 호스트는 게이트웨이 HTTP 서버에 의해 게이트웨이와 **동일한 포트**(기본 `18789`)에서 제공됩니다:
  - `/__openclaw__/canvas/`
  - `/__openclaw__/a2ui/`
    `gateway.auth`가 구성되고 게이트웨이가 루프백을 초과하여 바인딩되면 이러한 경로는 게이트웨이 인증으로 보호됩니다. 노드 클라이언트는 활성 WS 세션에 연결된 노드 범위 기능 URL을 사용합니다. [게이트웨이 구성](/gateway/configuration) (`canvasHost`, `gateway`) 참조.
- 원격 사용은 일반적으로 SSH 터널 또는 tailnet VPN입니다. [원격 액세스](/gateway/remote) 및 [검색](/gateway/discovery) 참조.
