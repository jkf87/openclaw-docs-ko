лӘ©м°Ё

OpenClawлҠ” мӢӨнҢЁлҘј л‘җ лӢЁкі„лЎң мІҳлҰ¬н•©лӢҲлӢӨ:

  1. нҳ„мһ¬ provider лӮҙм—җм„ңмқҳ Auth profile rotation.
  2. agents.defaults.model.fallbacksмқҳ лӢӨмқҢ лӘЁлҚёлЎңмқҳ лӘЁлҚё fallback.

мқҙ л¬ём„ңлҠ” лҹ°нғҖмһ„ к·ңм№ҷкіј к·ёкІғмқ„ л’·л°ӣм№Ён•ҳлҠ” лҚ°мқҙн„°лҘј м„ӨлӘ…н•©лӢҲлӢӨ.

лҹ°нғҖмһ„ н”ҢлЎңмҡ° вҖӢ

мқјл°ҳ н…ҚмҠӨнҠё мӢӨн–үмқҳ кІҪмҡ°, OpenClawлҠ” мқҙ мҲңм„ңлЎң нӣ„ліҙ(candidate)лҘј нҸүк°Җн•©лӢҲлӢӨ:

  1. нҳ„мһ¬ м„ нғқлҗң м„ём…ҳ лӘЁлҚё.
  2. кө¬м„ұлҗң agents.defaults.model.fallbacksлҘј мҲңм„ңлҢҖлЎң.
  3. мӢӨн–үмқҙ мҳӨлІ„лқјмқҙл“ңм—җм„ң мӢңмһ‘лҗҳм—Ҳмқ„ л•Ң л§Ҳм§Җл§үм—җ кө¬м„ұлҗң primary лӘЁлҚё.

к°Ғ нӣ„ліҙ лӮҙл¶Җм—җм„ң, OpenClawлҠ” лӢӨмқҢ лӘЁлҚё нӣ„ліҙлЎң м§„н–үн•ҳкё° м „м—җ auth-profile failoverлҘј лЁјм Җ мӢңлҸ„н•©лӢҲлӢӨ.

мғҒмң„ мҲҳмӨҖ мӢңнҖҖмҠӨ:

  1. нҷңм„ұ м„ём…ҳ лӘЁлҚёкіј auth-profile м„ нҳёлҸ„(preference)лҘј н•ҙм„қ(resolve)н•©лӢҲлӢӨ.
  2. лӘЁлҚё нӣ„ліҙ мІҙмқёмқ„ л№Ңл“ңн•©лӢҲлӢӨ.
  3. auth-profile rotation/cooldown к·ңм№ҷмңјлЎң нҳ„мһ¬ providerлҘј мӢңлҸ„н•©лӢҲлӢӨ.
  4. н•ҙлӢ№ providerк°Җ failover-worthy м—җлҹ¬лЎң мҶҢм§„лҗҳл©ҙ, лӢӨмқҢ лӘЁлҚё нӣ„ліҙлЎң мқҙлҸҷн•©лӢҲлӢӨ.
  5. лӢӨлҘё м„ём…ҳ readerк°Җ runnerк°Җ мӮ¬мҡ©н•ҳл ӨлҠ” кІғкіј лҸҷмқјн•ң provider/modelмқ„ ліҙлҸ„лЎқ мһ¬мӢңлҸ„к°Җ мӢңмһ‘лҗҳкё° м „м—җ м„ нғқлҗң fallback мҳӨлІ„лқјмқҙл“ңлҘј мҳҒмҶҚнҷ”(persist)н•©лӢҲлӢӨ.
  6. fallback нӣ„ліҙк°Җ мӢӨнҢЁн•ҳл©ҙ, н•ҙлӢ№ мӢӨнҢЁн•ң нӣ„ліҙмҷҖ м—¬м „нһҲ мқјм№ҳн•  л•Ңм—җ н•ңн•ҙ fallback мҶҢмң мқҳ м„ём…ҳ мҳӨлІ„лқјмқҙл“ң н•„л“ңл§Ң лЎӨл°ұн•©лӢҲлӢӨ.
  7. лӘЁл“  нӣ„ліҙк°Җ мӢӨнҢЁн•ҳл©ҙ, per-attempt мғҒм„ё м •ліҙмҷҖ м•Ңл Өм§„ кІҪмҡ° к°ҖмһҘ л№ лҘё cooldown л§ҢлЈҢ мӢңк°„мқ„ лӢҙмқҖ FallbackSummaryErrorлҘј throwн•©лӢҲлӢӨ.

мқҙлҠ” "м „мІҙ м„ём…ҳмқ„ м ҖмһҘн•ҳкі  ліөмӣҗ"н•ҳлҠ” кІғліҙлӢӨ мқҳлҸ„м ҒмңјлЎң лҚ” мўҒмқҖ лІ”мң„мһ…лӢҲлӢӨ. reply runnerлҠ” fallbackмқ„ мң„н•ҙ мһҗмӢ мқҙ мҶҢмң н•ң лӘЁлҚё м„ нғқ н•„л“ңл§Ң мҳҒмҶҚнҷ”н•©лӢҲлӢӨ:

  • providerOverride
  • modelOverride
  • authProfileOverride
  • authProfileOverrideSource
  • authProfileOverrideCompactionCount

мқҙлҠ” мӢӨнҢЁн•ң fallback мһ¬мӢңлҸ„к°Җ мӢңлҸ„к°Җ мӢӨн–үлҗҳлҠ” лҸҷм•Ҳ л°ңмғқн•ң мҲҳлҸҷ /model ліҖкІҪмқҙлӮҳ м„ём…ҳ rotation м—…лҚ°мқҙнҠёмҷҖ к°ҷмқҖ лҚ” мғҲлЎңмҡҙ кҙҖл Ё м—ҶлҠ” м„ём…ҳ mutationмқ„ лҚ®м–ҙм“°лҠ” кІғмқ„ л°©м§Җн•©лӢҲлӢӨ.

Auth м ҖмһҘмҶҢ (нӮӨ + OAuth) вҖӢ

OpenClawлҠ” API нӮӨмҷҖ OAuth нҶ нҒ° лӘЁл‘җм—җ лҢҖн•ҙ auth profileмқ„ мӮ¬мҡ©н•©лӢҲлӢӨ.

  • SecretмқҖ ~/.openclaw/agents/<agentId>/agent/auth-profiles.jsonм—җ мһҲмҠөлӢҲлӢӨ (л Ҳкұ°мӢң: ~/.openclaw/agent/auth-profiles.json).
  • лҹ°нғҖмһ„ auth-routing мғҒнғңлҠ” ~/.openclaw/agents/<agentId>/agent/auth-state.jsonм—җ мһҲмҠөлӢҲлӢӨ.
  • Config auth.profiles / auth.orderлҠ” metadata + routingл§Ңмһ…лӢҲлӢӨ (secret м—ҶмқҢ).
  • л Ҳкұ°мӢң import м „мҡ© OAuth нҢҢмқј: ~/.openclaw/credentials/oauth.json (мөңмҙҲ мӮ¬мҡ© мӢң auth-profiles.jsonмңјлЎң importлҗЁ).

мһҗм„ён•ң лӮҙмҡ©: /concepts/oauth

мһҗкІ© мҰқлӘ…(credential) нғҖмһ…:

  • type: "api_key" вҶ’ { provider, key }
  • type: "oauth" вҶ’ { provider, access, refresh, expires, email? } (мқјл¶Җ providerмқҳ кІҪмҡ° + projectId/enterpriseUrl)

Profile ID вҖӢ

OAuth лЎңк·ёмқёмқҖ м—¬лҹ¬ кі„м •мқҙ кіөмЎҙн•  мҲҳ мһҲлҸ„лЎқ кө¬лі„лҗҳлҠ” profileмқ„ мғқм„ұн•©лӢҲлӢӨ.

  • кё°ліёк°’: emailмқҙ м—Ҷмқ„ л•Ң provider:default.
  • emailмқҙ мһҲлҠ” OAuth: provider:<email> (мҳҲ: google-antigravity:user@gmail.com).

ProfileмқҖ ~/.openclaw/agents/<agentId>/agent/auth-profiles.jsonмқҳ profiles м•„лһҳм—җ мһҲмҠөлӢҲлӢӨ.

Rotation мҲңм„ң вҖӢ

providerм—җ м—¬лҹ¬ profileмқҙ мһҲмқ„ л•Ң, OpenClawлҠ” лӢӨмқҢкіј к°ҷмқҖ мҲңм„ңлҘј м„ нғқн•©лӢҲлӢӨ:

  1. лӘ…мӢңм Ғ config: auth.order[provider] (м„Өм •лҗң кІҪмҡ°).
  2. кө¬м„ұлҗң profile: providerлЎң н•„н„°л§Ғлҗң auth.profiles.
  3. м ҖмһҘлҗң profile: providerм—җ лҢҖн•ң auth-profiles.jsonмқҳ н•ӯлӘ©л“Ө.

лӘ…мӢңм Ғ мҲңм„ңк°Җ кө¬м„ұлҗҳм§Җ м•ҠмқҖ кІҪмҡ°, OpenClawлҠ” round-robin мҲңм„ңлҘј мӮ¬мҡ©н•©лӢҲлӢӨ:

  • Primary нӮӨ: profile нғҖмһ… (API нӮӨліҙлӢӨ OAuth лЁјм Җ).
  • Secondary нӮӨ: usageStats.lastUsed (к°Ғ нғҖмһ… лӮҙм—җм„ң к°ҖмһҘ мҳӨлһҳлҗң кІғ лЁјм Җ).
  • Cooldown/л№„нҷңм„ұнҷ”лҗң profileмқҖ к°ҖмһҘ л№ лҘё л§ҢлЈҢ мҲңмңјлЎң лҒқмңјлЎң мқҙлҸҷлҗ©лӢҲлӢӨ.

м„ём…ҳ stickiness (cache-friendly) вҖӢ

OpenClawлҠ” provider мәҗмӢңлҘј л”°лң»н•ҳкІҢ мң м§Җн•ҳкё° мң„н•ҙ м„ нғқлҗң auth profileмқ„ м„ём…ҳлӢ№ н•Җ(pin)н•©лӢҲлӢӨ. л§Ө мҡ”мІӯл§ҲлӢӨ rotationн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ. н•Җлҗң profileмқҖ лӢӨмқҢк№Ңм§Җ мһ¬мӮ¬мҡ©лҗ©лӢҲлӢӨ:

  • м„ём…ҳмқҙ лҰ¬м…ӢлҗҳлҠ” кІҪмҡ° (/new / /reset)
  • compactionмқҙ мҷ„лЈҢлҗҳлҠ” кІҪмҡ° (compaction м№ҙмҡҙнҠё мҰқк°Җ)
  • profileмқҙ cooldown/л№„нҷңм„ұнҷ” мғҒнғңмқё кІҪмҡ°

/model вҖҰ@<profileId>лҘј нҶөн•ң мҲҳлҸҷ м„ нғқмқҖ н•ҙлӢ№ м„ём…ҳм—җ лҢҖн•ң мӮ¬мҡ©мһҗ мҳӨлІ„лқјмқҙл“ңлҘј м„Өм •н•ҳл©° мғҲ м„ём…ҳмқҙ мӢңмһ‘лҗ  л•Ңк№Ңм§Җ мһҗлҸҷ rotationлҗҳм§Җ м•ҠмҠөлӢҲлӢӨ.

мһҗлҸҷмңјлЎң н•Җлҗң profile (м„ём…ҳ лқјмҡ°н„°м—җ мқҳн•ҙ м„ нғқлҗЁ)мқҖ **м„ нҳёлҸ„(preference)**лЎң м·Ёкёүлҗ©лӢҲлӢӨ: лЁјм Җ мӢңлҸ„лҗҳм§Җл§Ң, OpenClawлҠ” rate limit/timeout мӢң лӢӨлҘё profileлЎң rotationн•  мҲҳ мһҲмҠөлӢҲлӢӨ. мӮ¬мҡ©мһҗ н•Җлҗң profileмқҖ н•ҙлӢ№ profileм—җ мһ кёҙ мұ„лЎң мң м§Җлҗ©лӢҲлӢӨ; мӢӨнҢЁн•ҳкі  лӘЁлҚё fallbackмқҙ кө¬м„ұлҗң кІҪмҡ°, OpenClawлҠ” profileмқ„ м „нҷҳн•ҳлҠ” лҢҖмӢ  лӢӨмқҢ лӘЁлҚёлЎң мқҙлҸҷн•©лӢҲлӢӨ.

OAuthк°Җ "мӮ¬лқјм§„ кІғмІҳлҹј ліҙмқҙлҠ”" мқҙмң  вҖӢ

лҸҷмқјн•ң providerм—җ лҢҖн•ҙ OAuth profileкіј API нӮӨ profile лӘЁл‘җлҘј к°Җм§Җкі  мһҲлҠ” кІҪмҡ°, н•Җлҗҳм§Җ м•ҠлҠ” н•ң round-robinмқҙ л©”мӢңм§Җ к°„м—җ л‘ҳ мӮ¬мқҙлҘј м „нҷҳн•  мҲҳ мһҲмҠөлӢҲлӢӨ. лӢЁмқј profileмқ„ к°•м ңн•ҳл Өл©ҙ:

  • auth.order[provider] = ["provider:profileId"]лЎң н•Җн•ҳкұ°лӮҳ,
  • profile мҳӨлІ„лқјмқҙл“ңк°Җ мһҲлҠ” /model вҖҰмқ„ нҶөн•ң per-session мҳӨлІ„лқјмқҙл“ңлҘј мӮ¬мҡ©н•ҳмӢӯмӢңмҳӨ (UI/chat surfaceм—җм„ң м§ҖмӣҗлҗҳлҠ” кІҪмҡ°).

Cooldown вҖӢ

profileмқҙ auth/rate-limit м—җлҹ¬ (лҳҗлҠ” rate limitingмІҳлҹј ліҙмқҙлҠ” timeout)лЎң мӢӨнҢЁн•ҳл©ҙ, OpenClawлҠ” к·ёкІғмқ„ cooldownмңјлЎң н‘ңмӢңн•ҳкі  лӢӨмқҢ profileлЎң мқҙлҸҷн•©лӢҲлӢӨ. н•ҙлӢ№ rate-limit лІ„нӮ·мқҖ лӢЁмҲңн•ң 429ліҙлӢӨ лҚ” кҙ‘лІ”мң„н•©лӢҲлӢӨ: Too many concurrent requests, ThrottlingException, concurrency limit reached, workers_ai ... quota limit exceeded, throttled, resource exhausted, к·ёлҰ¬кі  weekly/monthly limit reachedмҷҖ к°ҷмқҖ мЈјкё°м Ғ usage-window н•ңлҸ„мҷҖ к°ҷмқҖ provider л©”мӢңм§ҖлҸ„ нҸ¬н•Ён•©лӢҲлӢӨ. Format/invalid-request м—җлҹ¬ (мҳҲ: Cloud Code Assist tool call ID кІҖмҰқ мӢӨнҢЁ)лҠ” failover-worthyлЎң м·Ёкёүлҗҳл©° лҸҷмқјн•ң cooldownмқ„ мӮ¬мҡ©н•©лӢҲлӢӨ. Billing л№„нҷңм„ұнҷ” вҖӢ

Billing/credit мӢӨнҢЁ (мҳҲ: "insufficient credits" / "credit balance too low")лҠ” failover-worthyлЎң м·Ёкёүлҗҳм§Җл§Ң, мқјл°ҳм ҒмңјлЎң transientн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ. м§§мқҖ cooldown лҢҖмӢ , OpenClawлҠ” profileмқ„ л№„нҷңм„ұнҷ”лЎң н‘ңмӢңн•ҳкі  (лҚ” кёҙ backoffмҷҖ н•Ёк»ҳ) лӢӨмқҢ profile/providerлЎң rotationн•©лӢҲлӢӨ.

лӘЁл“  billing-shaped мқ‘лӢөмқҙ 402лҠ” м•„лӢҲл©°, лӘЁл“  HTTP 402к°Җ м—¬кё°м—җ н•ҙлӢ№н•ҳлҠ” кІғлҸ„ м•„лӢҷлӢҲлӢӨ. OpenClawлҠ” providerк°Җ 401мқҙлӮҳ 403мқ„ лҢҖмӢ  л°ҳнҷҳн•ҳлҚ”лқјлҸ„ лӘ…мӢңм Ғмқё billing н…ҚмҠӨнҠёлҘј billing л Ҳмқём—җ мң м§Җн•ҳм§Җл§Ң, provider-specific matcherлҠ” к·ёкІғмқ„ мҶҢмң н•ң providerлЎң лІ”мң„к°Җ м§Җм •лҗң мұ„лЎң мң м§Җлҗ©лӢҲлӢӨ (мҳҲ: OpenRouter 403 Key limit exceeded). н•ңнҺё мһ„мӢң 402 usage-window л°Ҹ organization/workspace spend-limit м—җлҹ¬лҠ” л©”мӢңм§Җк°Җ retryableн•ҳкІҢ ліҙмқј л•Ң rate_limitмңјлЎң л¶„лҘҳлҗ©лӢҲлӢӨ (мҳҲ: weekly usage limit exhausted, daily limit reached, resets tomorrow, organization spending limit exceeded). мқҙл“ӨмқҖ кёҙ billing-disable кІҪлЎң лҢҖмӢ  м§§мқҖ cooldown/failover кІҪлЎңм—җ мң м§Җлҗ©лӢҲлӢӨ.

  • profileмқҙ 24мӢңк°„ лҸҷм•Ҳ мӢӨнҢЁн•ҳм§Җ м•Ҡмңјл©ҙ backoff м№ҙмҡҙн„°к°Җ лҰ¬м…Ӣлҗ©лӢҲлӢӨ (кө¬м„ұ к°ҖлҠҘ).

лӘЁлҚё fallback вҖӢ

providerм—җ лҢҖн•ң лӘЁл“  profileмқҙ мӢӨнҢЁн•ҳл©ҙ, OpenClawлҠ” agents.defaults.model.fallbacksмқҳ лӢӨмқҢ лӘЁлҚёлЎң мқҙлҸҷн•©лӢҲлӢӨ. мқҙлҠ” auth мӢӨнҢЁ, rate limit, к·ёлҰ¬кі  profile rotationмқҙ мҶҢм§„лҗң timeoutм—җ м Ғмҡ©лҗ©лӢҲлӢӨ (лӢӨлҘё м—җлҹ¬лҠ” fallbackмқ„ м§„н–үмӢңнӮӨм§Җ м•ҠмҠөлӢҲлӢӨ).

Overloaded л°Ҹ rate-limit м—җлҹ¬лҠ” billing cooldownліҙлӢӨ лҚ” кіөкІ©м ҒмңјлЎң мІҳлҰ¬лҗ©лӢҲлӢӨ. кё°ліём ҒмңјлЎң, OpenClawлҠ” н•ң лІҲмқҳ same-provider auth-profile мһ¬мӢңлҸ„лҘј н—Ҳмҡ©н•ң лӢӨмқҢ, кё°лӢӨлҰ¬м§Җ м•Ҡкі  лӢӨмқҢ кө¬м„ұлҗң лӘЁлҚё fallbackмңјлЎң м „нҷҳн•©лӢҲлӢӨ. ModelNotReadyExceptionкіј к°ҷмқҖ provider-busy мӢ нҳёлҠ” н•ҙлӢ№ overloaded лІ„нӮ·м—җ л“Өм–ҙк°‘лӢҲлӢӨ. auth.cooldowns.overloadedProfileRotations, auth.cooldowns.overloadedBackoffMs, auth.cooldowns.rateLimitedProfileRotationsлЎң мқҙлҘј мЎ°м •н•ҳмӢӯмӢңмҳӨ.

мӢӨн–үмқҙ лӘЁлҚё мҳӨлІ„лқјмқҙл“ңлЎң мӢңмһ‘лҗҳлҠ” кІҪмҡ° (hook лҳҗлҠ” CLI), fallbackмқҖ кө¬м„ұлҗң fallbackмқ„ мӢңлҸ„н•ң нӣ„ м—¬м „нһҲ agents.defaults.model.primaryм—җм„ң мў…лЈҢлҗ©лӢҲлӢӨ.

нӣ„ліҙ мІҙмқё к·ңм№ҷ вҖӢ

OpenClawлҠ” нҳ„мһ¬ мҡ”мІӯлҗң provider/modelкіј кө¬м„ұлҗң fallbackмңјлЎңл¶Җн„° нӣ„ліҙ лҰ¬мҠӨнҠёлҘј л№Ңл“ңн•©лӢҲлӢӨ.

к·ңм№ҷ:

  • мҡ”мІӯлҗң лӘЁлҚёмқҙ н•ӯмғҒ мІ« лІҲм§ёмһ…лӢҲлӢӨ.
  • лӘ…мӢңм ҒмңјлЎң кө¬м„ұлҗң fallbackмқҖ мӨ‘ліө м ңкұ°лҗҳм§Җл§Ң лӘЁлҚё allowlistлЎң н•„н„°л§Ғлҗҳм§ҖлҠ” м•ҠмҠөлӢҲлӢӨ. лӘ…мӢңм Ғ operator мқҳлҸ„лЎң м·Ёкёүлҗ©лӢҲлӢӨ.
  • нҳ„мһ¬ мӢӨн–үмқҙ мқҙлҜё лҸҷмқјн•ң provider нҢЁл°ҖлҰ¬мқҳ кө¬м„ұлҗң fallbackм—җ мһҲлӢӨл©ҙ, OpenClawлҠ” м „мІҙ кө¬м„ұлҗң мІҙмқёмқ„ кі„мҶҚ мӮ¬мҡ©н•©лӢҲлӢӨ.
  • нҳ„мһ¬ мӢӨн–үмқҙ configмҷҖ лӢӨлҘё providerм—җ мһҲкі  к·ё нҳ„мһ¬ лӘЁлҚёмқҙ мқҙлҜё кө¬м„ұлҗң fallback мІҙмқёмқҳ мқјл¶Җк°Җ м•„лӢҲлқјл©ҙ, OpenClawлҠ” лӢӨлҘё providerмқҳ кҙҖл Ё м—ҶлҠ” кө¬м„ұлҗң fallbackмқ„ м¶”к°Җн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ.
  • мӢӨн–үмқҙ мҳӨлІ„лқјмқҙл“ңм—җм„ң мӢңмһ‘лҗң кІҪмҡ°, мқҙм „ нӣ„ліҙк°Җ мҶҢм§„лҗң нӣ„ мІҙмқёмқҙ м •мғҒ кё°ліёк°’мңјлЎң лҸҢм•„мҷҖ м•Ҳм°©н•  мҲҳ мһҲлҸ„лЎқ кө¬м„ұлҗң primaryк°Җ лҒқм—җ м¶”к°Җлҗ©лӢҲлӢӨ.

м–ҙл–Ө м—җлҹ¬к°Җ fallbackмқ„ м§„н–үмӢңнӮӨлҠ”к°Җ вҖӢ

  • rate limit л°Ҹ cooldown мҶҢм§„

  • м—¬м „нһҲ лӮЁмқҖ нӣ„ліҙк°Җ мһҲмқ„ л•Ңмқҳ лӢӨлҘё мқёмӢқлҗҳм§Җ м•ҠмқҖ м—җлҹ¬

Cooldown skip лҢҖ probe лҸҷмһ‘ вҖӢ

providerм—җ лҢҖн•ң лӘЁл“  auth profileмқҙ мқҙлҜё cooldown мғҒнғңмқј л•Ң, OpenClawлҠ” н•ҙлӢ№ providerлҘј мҳҒмӣҗнһҲ мһҗлҸҷмңјлЎң skipн•ҳм§ҖлҠ” м•ҠмҠөлӢҲлӢӨ. per-candidate кІ°м •мқ„ лӮҙлҰҪлӢҲлӢӨ:

  • Billing л№„нҷңм„ұнҷ”лҠ” мқјл°ҳм ҒмңјлЎң skipн•ҳм§Җл§Ң, primary нӣ„ліҙлҠ” м—¬м „нһҲ throttle мғҒм—җм„ң probeлҗ  мҲҳ мһҲм–ҙ мһ¬мӢңмһ‘ м—Ҷмқҙ ліөкө¬к°Җ к°ҖлҠҘн•©лӢҲлӢӨ.
  • primary нӣ„ліҙлҠ” per-provider throttleкіј н•Ёк»ҳ cooldown л§ҢлЈҢ к·јмІҳм—җм„ң probeлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.
  • Same-provider fallback siblingмқҖ мӢӨнҢЁк°Җ transientн•ҙ ліҙмқҙлҠ” кІҪмҡ° (rate_limit, overloaded, лҳҗлҠ” unknown) cooldownм—җлҸ„ л¶Ҳкө¬н•ҳкі  мӢңлҸ„лҗ  мҲҳ мһҲмҠөлӢҲлӢӨ. мқҙлҠ” rate limitмқҙ model-scopedмқҙкі  sibling лӘЁлҚёмқҙ м—¬м „нһҲ мҰүмӢң ліөкө¬лҗ  мҲҳ мһҲмқ„ л•Ң нҠ№нһҲ кҙҖл Ёмқҙ мһҲмҠөлӢҲлӢӨ.
  • Transient cooldown probeлҠ” лӢЁмқј providerк°Җ cross-provider fallbackмқ„ м§Җм—°мӢңнӮӨм§Җ м•ҠлҸ„лЎқ fallback мӢӨн–үлӢ№ providerлӢ№ н•ҳлӮҳлЎң м ңн•ңлҗ©лӢҲлӢӨ.
