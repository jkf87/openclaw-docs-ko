# 구성 예시

아래 예시는 현재 구성 스키마에 맞게 정렬되어 있습니다. 상세한 레퍼런스와 필드별 참고는 [구성](/gateway/configuration)을 참조하십시오.

## 빠른 시작

### 절대 최소

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

`~/.openclaw/openclaw.json`에 저장하면 해당 번호에서 봇에 DM을 보낼 수 있습니다.

### 권장 스타터

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-6" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## 일반적인 패턴

### 하나의 재정의가 있는 공유 스킬 기준

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      skills: ["github", "weather"],
    },
    list: [
      { id: "main", default: true },
      { id: "docs", workspace: "~/.openclaw/workspace-docs", skills: ["docs-search"] },
    ],
  },
}
```

- `agents.defaults.skills`는 공유 기준입니다.
- `agents.list[].skills`는 하나의 에이전트에 대한 해당 기준을 대체합니다.
- 에이전트가 스킬을 보지 않아야 할 때 `skills: []`를 사용하십시오.

### 다중 플랫폼 설정

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### 보안 DM 모드 (공유 인박스 / 다중 사용자 DM)

두 명 이상이 봇에 DM을 보낼 수 있는 경우(`allowFrom`의 여러 항목, 여러 사람에 대한 페어링 승인, 또는 `dmPolicy: "open"`), **보안 DM 모드**를 활성화하여 서로 다른 발신자의 DM이 기본적으로 하나의 컨텍스트를 공유하지 않도록 하십시오:

```json5
{
  // 보안 DM 모드 (다중 사용자 또는 민감한 DM 에이전트에 권장)
  session: { dmScope: "per-channel-peer" },

  channels: {
    // 예시: WhatsApp 다중 사용자 인박스
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // 예시: Discord 다중 사용자 인박스
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

Discord/Slack/Google Chat/Microsoft Teams/Mattermost/IRC의 경우, 발신자 권한 부여는 기본적으로 ID 우선입니다.
해당 위험을 명시적으로 수용하는 경우에만 각 채널의 `dangerouslyAllowNameMatching: true`로 직접 변경 가능한 이름/이메일/닉 매칭을 활성화하십시오.

## 팁

- `dmPolicy: "open"`을 설정하는 경우, 매칭하는 `allowFrom` 목록에 `"*"`가 포함되어야 합니다.
- 프로바이더 ID는 다릅니다(전화번호, 사용자 ID, 채널 ID). 형식을 확인하려면 프로바이더 문서를 사용하십시오.
- 나중에 추가할 선택적 섹션: `web`, `browser`, `ui`, `discovery`, `canvasHost`, `talk`, `signal`, `imessage`.
- 더 깊은 설정 참고는 [프로바이더](/providers/) 및 [문제 해결](/gateway/troubleshooting)을 참조하십시오.
