# 마크다운 포맷팅

OpenClaw는 채널별 출력을 렌더링하기 전에 공유 중간 표현(IR)으로 변환하여 아웃바운드 마크다운을 포맷팅합니다. IR은 스타일/링크 스팬을 전달하면서 소스 텍스트를 그대로 유지하여 청킹과 렌더링이 채널 간에 일관되게 유지될 수 있습니다.

## 목표

- **일관성:** 하나의 파싱 단계, 여러 렌더러.
- **안전한 청킹:** 인라인 포맷팅이 청크 간에 분리되지 않도록 렌더링 전에 텍스트를 분할합니다.
- **채널 적합성:** 마크다운을 다시 파싱하지 않고 동일한 IR을 Slack mrkdwn, Telegram HTML, Signal 스타일 범위에 매핑합니다.

## 파이프라인

1. **마크다운 -> IR 파싱**
   - IR은 일반 텍스트에 스타일 스팬 (굵기/기울임/취소선/코드/스포일러)과 링크 스팬이 더해진 것입니다.
   - 오프셋은 UTF-16 코드 단위이므로 Signal 스타일 범위가 API와 정렬됩니다.
   - 테이블은 채널이 테이블 변환을 선택할 때만 파싱됩니다.
2. **IR 청킹 (포맷 우선)**
   - 청킹은 렌더링 전에 IR 텍스트에서 발생합니다.
   - 인라인 포맷팅은 청크 간에 분리되지 않습니다; 스팬은 청크별로 분할됩니다.
3. **채널별 렌더링**
   - **Slack:** mrkdwn 토큰 (굵기/기울임/취소선/코드), 링크는 `<url|label>`.
   - **Telegram:** HTML 태그 (`<b>`, `<i>`, `<s>`, `<code>`, `<pre><code>`, `<a href>`).
   - **Signal:** 일반 텍스트 + `text-style` 범위; 레이블이 URL과 다른 경우 링크는 `label (url)`이 됩니다.

## IR 예시

입력 마크다운:

```markdown
Hello **world** — see [docs](https://docs.openclaw.ai).
```

IR (개략적):

```json
{
  "text": "Hello world — see docs.",
  "styles": [{ "start": 6, "end": 11, "style": "bold" }],
  "links": [{ "start": 19, "end": 23, "href": "https://docs.openclaw.ai" }]
}
```

## 사용 위치

- Slack, Telegram, Signal 아웃바운드 어댑터가 IR에서 렌더링합니다.
- 다른 채널 (WhatsApp, iMessage, Microsoft Teams, Discord)은 여전히 일반 텍스트 또는 자체 포맷팅 규칙을 사용하며, 활성화된 경우 청킹 전에 마크다운 테이블 변환이 적용됩니다.

## 테이블 처리

마크다운 테이블은 채팅 클라이언트 간에 일관적으로 지원되지 않습니다. 채널당 (및 계정당) 변환을 제어하려면 `markdown.tables`를 사용하십시오.

- `code`: 테이블을 코드 블록으로 렌더링합니다 (대부분의 채널에서 기본값).
- `bullets`: 각 행을 글머리 기호로 변환합니다 (Signal + WhatsApp 기본값).
- `off`: 테이블 파싱 및 변환을 비활성화합니다; 원시 테이블 텍스트가 그대로 통과합니다.

구성 키:

```yaml
channels:
  discord:
    markdown:
      tables: code
    accounts:
      work:
        markdown:
          tables: off
```

## 청킹 규칙

- 청크 한도는 채널 어댑터/구성에서 오며 IR 텍스트에 적용됩니다.
- 코드 펜스는 채널이 올바르게 렌더링할 수 있도록 후행 줄바꿈과 함께 단일 블록으로 보존됩니다.
- 목록 접두사 및 인용구 접두사는 IR 텍스트의 일부이므로 청킹이 접두사 중간에 분리되지 않습니다.
- 인라인 스타일 (굵기/기울임/취소선/인라인 코드/스포일러)은 청크 간에 절대 분리되지 않습니다; 렌더러는 각 청크 내에서 스타일을 다시 엽니다.

채널 간 청킹 동작에 대한 자세한 내용은 [스트리밍 + 청킹](/concepts/streaming)을 참조하십시오.

## 링크 정책

- **Slack:** `[label](url)` -> `<url|label>`; 베어 URL은 베어로 유지됩니다. 이중 링크를 방지하기 위해 파싱 중에 자동 링크가 비활성화됩니다.
- **Telegram:** `[label](url)` -> `<a href="url">label</a>` (HTML 파스 모드).
- **Signal:** `[label](url)` -> 레이블이 URL과 일치하지 않으면 `label (url)`.

## 스포일러

스포일러 마커 (`||spoiler||`)는 Signal에서만 파싱되어 SPOILER 스타일 범위에 매핑됩니다. 다른 채널에서는 일반 텍스트로 처리됩니다.

## 채널 포매터 추가 또는 업데이트 방법

1. **한 번 파싱:** 채널에 적합한 옵션 (자동 링크, 제목 스타일, 인용구 접두사)으로 공유 `markdownToIR(...)` 헬퍼를 사용하십시오.
2. **렌더링:** `renderMarkdownWithMarkers(...)`와 스타일 마커 맵 (또는 Signal 스타일 범위)으로 렌더러를 구현하십시오.
3. **청킹:** 렌더링 전에 `chunkMarkdownIR(...)`를 호출하고 각 청크를 렌더링하십시오.
4. **어댑터 연결:** 새 청커와 렌더러를 사용하도록 채널 아웃바운드 어댑터를 업데이트하십시오.
5. **테스트:** 포맷 테스트와 채널이 청킹을 사용하는 경우 아웃바운드 전달 테스트를 추가하거나 업데이트하십시오.

## 일반적인 주의사항

- Slack 꺾쇠 괄호 토큰 (`<@U123>`, `<#C123>`, `<https://...>`)은 보존해야 합니다; 원시 HTML을 안전하게 이스케이프하십시오.
- Telegram HTML은 손상된 마크업을 방지하기 위해 태그 외부의 텍스트를 이스케이프해야 합니다.
- Signal 스타일 범위는 UTF-16 오프셋에 의존합니다; 코드 포인트 오프셋을 사용하지 마십시오.
- 닫는 마커가 자체 줄에 위치하도록 펜스드 코드 블록에 후행 줄바꿈을 보존하십시오.
